# Sekhar AI OS Self-Improvement Protocol

The AI may propose improvements, generate patches, and run tests.

The AI may not silently modify:

- safety rules
- approval rules
- credential handling
- destructive command restrictions
- logs
- rollback system

Every self-update must include:

1. Reason for change
2. Files changed
3. Test result
4. Risk level
5. Rollback plan
6. User approval
