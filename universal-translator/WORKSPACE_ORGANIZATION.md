# Workspace Organization

Universal Translator is a standalone project. It must install, run, test, and
package from this folder without requiring any sibling project.

## Sync Rule

Source code, scripts, documentation, tests, and configuration for Universal
Translator live in this project. Generated runtime data stays local and ignored.

## Standalone Rule

Do not mirror other project trees into `universal-translator/tools/`. Do not
reference a sibling project as a runtime dependency. If Universal Translator
needs an improvement note, record it in `docs/IMPROVEMENT_REGISTRY.md`.

## Excluded Generated Data

The following categories are intentionally excluded:

- `.ult-runtime/`
- `.next/`
- `node_modules/`
- `dist-exe/`
- generated logs, model caches, and packaged app outputs

These are runtime artifacts, downloaded binaries, caches, or evidence output.
They should be regenerated by running Universal Translator locally.
