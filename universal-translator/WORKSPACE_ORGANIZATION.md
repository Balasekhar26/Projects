# Workspace Organization

The local Balu Projects workspace now contains the GitHub repository at:

```text
universal-translator/
```

The active companion projects are canonical top-level folders:

```text
ai-cyber-shield/
universal-ai/
pcb-doctor/
musical-keyboard/
```

## Sync Rule

Source code, scripts, documentation, tests, and configuration from the local workspace should live in the repo. Generated runtime data should stay local and ignored.

## Canonical Project Copies

Use these Balu Projects root folders:

```text
ai-cyber-shield/
universal-ai/
pcb-doctor/
musical-keyboard/
universal-translator/
```

Do not mirror project trees back into `universal-translator/tools/`. That old layout made the translator repo look like a catch-all workspace and reintroduced stale paths.

## Excluded Generated Data

The following categories are intentionally excluded:

- `universal-ai/.local/`
- `universal-ai/.installed.flag`
- `universal-ai/bin/ollama`
- `universal-ai/logs/`
- `universal-ai/memory/`
- `ai-cyber-shield/runtime/logs/`
- `ai-cyber-shield/runtime/evidence/`
- `ai-cyber-shield/runtime/quarantine/`
- generated `ai-cyber-shield/reports/incident-*.md`

These are runtime artifacts, downloaded binaries, caches, or evidence output. They should be regenerated by running each tool locally.
